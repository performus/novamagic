# NOVAMAGIC — Dubai Wind. Спецификация лендинга

> Премиальный лендинг парфюмированного концентрата для стирки. Эстетика нишевой парфюмерии: светлый кремовый фон, глубокий зелёный, золото, рендеры бутылки. Финальная фишка — бутылка, вращающаяся по скроллу (image-секвенция).

**Источник истины по дизайну:** Figma `U2exszq3xuSZRg788b89ox`, фрейм `114:2` («О нас», desktop 1920×7911).
**Принцип работы:** реализуем строго по фазам (см. §8). Не переходим к следующей фазе, пока предыдущая не закрыта и не проверена на localhost.

---

## 1. Цель и принципы

- Продаём **ощущение**, не утилиту. Тон — нишевая парфюмерия (верх/сердце/база, шлейф, «звучание»). Без восклицательных знаков и превосходных степеней.
- Дизайн «дорогой и тихий»: воздух, крупная типографика, аккуратные рендеры, матовое стекло, мягкие тени.
- Картинки и рендеры — главные герои. Вёрстка их обслуживает, а не наоборот.
- Производительность не приносится в жертву: секвенция бутылки оптимизируется (WebP, preload, canvas), никаких тяжёлых GIF/видео.

## 2. Технический стек (рекомендация)

| Слой | Выбор | Почему |
|---|---|---|
| Сборка | **Vite + React + TypeScript** | Мгновенный HMR для поблочной работы локально, минимум магии |
| Стили | **CSS Modules + `tokens.css`** (CSS-переменные) | Полный контроль над «дорогим» дизайном, без templated-вида от utility-классов |
| Скролл | **Lenis** (smooth scroll) | Плавность premium-уровня, дружит с ScrollTrigger |
| Анимации | **GSAP + ScrollTrigger** | Индустриальный стандарт для pin + scrub секвенций (Apple-style) |
| Секвенция бутылки | `<canvas>` + preloaded WebP-кадры | Самый производительный способ покадрового скролла |
| Шрифт | **Work Sans** (Google Fonts: 400/500/600) | Подтверждён в макете |

> Альтернатива: Next.js (static export) — если нужен SSR/SEO «из коробки». Для одностраничника Vite проще и быстрее.

**Анализ референсов (подтверждает стек):**
- **ethicallifeworld.com** — главный референс по анимации. Не Webflow/Framer, а hand-coded WordPress-тема. Герой — **scroll-scrubbed секвенция WebP** (`img/scrub-N/webp/S1_00000.webp … S1_00160.webp`, ~160 кадров, несколько секвенций `scrub-1…scrub-8` по странице). Это ровно наша техника — повторяем 1:1 в коде через GSAP+canvas.
- **eathungrytiger.com**, **more-nutrition.webflow.io** — Webflow. Отвечают за «ощущение»: sticky-секции, мягкие reveal-появления, поп-ин иконок/ингредиентов. Всё это закрывается GSAP ScrollTrigger без Webflow.

Вывод: **Webflow/Framer не нужны.** Для скролл-секвенций код даёт больше контроля и лучшую производительность. Анимационные либы (`gsap`, `lenis`) ставим **только в Фазе 2**, чтобы не мешали статике.

## 5. Адаптив

- **База:** desktop 1920 (по фрейму 114:2).
- **Брейкпоинты:** `≥1440` (laptop), `768–1439` (tablet — перестроить многоколоночные блоки в 1–2 колонки), `≤767` (mobile, цель 390 — есть отдельный mobile-wireframe в Figma как референс).
- Сетка: контент в контейнере `--container-max`, по краям воздух. Большинство блоков — flex/grid по центру.
- На мобайле: пирамида ароматов и фичи-грид складываются в вертикальный стек; рендеры центрируются.

## 7. Контракт на раскадровку бутылки (делается параллельно)

> Это нужно, чтобы твои кадры из nano banana / 3D точно встали в код без переделок. Зафиксируй параметры **до** генерации.

**Что снимаем:** полный оборот бутылки 360° (B1 Hero — основной; опционально B5/B9).

| Параметр | Значение |
|---|---|
| Кол-во кадров | **120–150** (плавно). Референс ethicallifeworld использует ~160 кадров на секвенцию. Минимум 90 |
| Разрешение | **1500×1500** px, бутылка по центру, единый масштаб во всех кадрах |
| Формат | **WebP** (quality ~82). Если нужна прозрачность — PNG, потом конверт в WebP |
| Фон | прозрачный **или** кремовый `#F7F2E7` (единый во всех кадрах) |
| Именование | `bottle_0001.webp … bottle_0120.webp` (zero-pad 4 знака, строго по порядку оборота) |
| Папка | `public/sequences/hero-bottle/` |
| Вес всей секвенции | цель **< 6–8 МБ** суммарно (оптимизировать) |
| Манифест | `manifest.json`: `{ "frameCount": 120, "pattern": "bottle_%04d.webp", "width": 1500, "height": 1500 }` |

**Критично:** центр и масштаб бутылки **не должны прыгать** между кадрами (иначе при скролле она будет «дрожать»). Свет/блики — консистентные. Этикетка читаемо проходит фронтальный кадр.

**Как это будет работать в коде (Фаза 2):** компонент `HeroBottleSequence` пинит секцию, предзагружает все кадры, и через `ScrollTrigger` со `scrub` маппит прогресс скролла в индекс кадра, отрисовывая текущий кадр на `<canvas>`. Никаких CSS-трансформаций — только смена кадра.

## 8. Фазы реализации (с точками остановки)

### Фаза 0 — Каркас (1 заход)
1. `npm create vite@latest novamagic -- --template react-ts`
2. Структура папок из §3, `tokens.css` из §4, reset + globals, подключить Work Sans.
3. `App.tsx` рендерит **пустые секции-заглушки** B0…B10 (просто блоки нужной высоты с подписью).
4. Контейнер, базовая сетка, фон `--bg-cream`.
5. **Запуск `npm run dev` → проверяем localhost: видим вертикальную ленту из 11 пустых блоков.**
   ⛔️ **STOP. Показываю тебе, согласуем, только потом дальше.**

### Фаза 1 — Статика, блок за блоком
Для каждого блока B0→B10 (по одному за раз):
1. `get_design_context` по node ID блока → точные размеры/цвета/текст.
2. Свёрстать pixel-match по десктопу, реальная копирайт-копия (не плейсхолдеры).
3. Подключить статичные рендеры/ингредиенты как обычные `<img>`.
4. Адаптив: laptop → tablet → mobile.
5. **Проверка на localhost → STOP после каждого блока, согласуем, идём к следующему.**

Итог Фазы 1: полный статичный лендинг на localhost, без единой анимации. Бутылка — статичный рендер.

### Фаза 2 — Анимационный слой (когда готова раскадровка)
1. Ставим `gsap`, `lenis`. Подключаем Lenis smooth scroll (`useLenis`).
2. Entrance-reveals (мягкие fade/slide-up по `ScrollTrigger` на заголовках/карточках).
3. Sticky-header: фон появляется при скролле.
4. **Главное:** `HeroBottleSequence` (canvas + scrub) на B1 по §7. Пин секции, маппинг скролл→кадр.
5. Опционально — вращение/параллакс рендеров в B5/B9.
6. Тюнинг производительности: preload, `will-change`, проверка на слабом железе.

## 10. Зафиксированные решения

1. **Стек:** Vite + React + TypeScript + GSAP ScrollTrigger + Lenis + canvas-секвенция. Подтверждён анализом референсов (§2).
2. **Desktop-first** от фрейма 114:2. Мобильный wireframe (390) — референс для ≤767.
3. Узлы «главная»/«акции» за фреймом — **мусор, игнорируем.**
4. **Только RU** на этом этапе. Английская версия — отдельным этапом после готового сайта. На Фазе 1 не внедряем i18n, но структуру держим i18n-дружелюбной (текст не хардкодим в разметку намертво — выносим в `content.ts` на блок, чтобы потом легко продублировать EN). Переключатель «EN \ RU» в хедере пока статичный/декоративный.
5. **OZON/YANDEX** — внешние ссылки на товар (`https://ozon.ru/t/C6s3tyi` и аналог Яндекса), открываются в новой вкладке.
